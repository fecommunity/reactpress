# Contact

We welcome questions, bug reports, feature ideas, and contributions from the ReactPress community.

## Support channels

| Channel | Best for | Link |
| --- | --- | --- |
| **GitHub Issues** | Bugs, reproducible problems | [github.com/fecommunity/reactpress/issues](https://github.com/fecommunity/reactpress/issues) |
| **GitHub Discussions** | Questions, ideas, show-and-tell | [github.com/fecommunity/reactpress/discussions](https://github.com/fecommunity/reactpress/discussions) |
| **Email** | Private inquiries, security reports | [admin@gaoredu.com](mailto:admin@gaoredu.com) |

## Before opening an issue

1. Run `reactpress doctor` and `reactpress logs` on your installation
2. Search [existing issues](https://github.com/fecommunity/reactpress/issues) for duplicates
3. Read the [FAQ](/docs/reference/faq) and [Troubleshooting](/docs/reference/troubleshooting) guides
4. Include Node.js version, OS, ReactPress version, and relevant log output

See [How To Ask Questions The Smart Way](https://github.com/ryanhanwu/How-To-Ask-Questions-The-Smart-Way) for tips that help maintainers respond faster.

## Security reports

If you discover a security vulnerability, please **do not** open a public issue. Email [admin@gaoredu.com](mailto:admin@gaoredu.com) with details and allow reasonable time for a fix before public disclosure.

## Contributing

Pull requests are welcome on [GitHub](https://github.com/fecommunity/reactpress/pulls). Documentation edits can be submitted directly from any doc page via **Edit this page**.
